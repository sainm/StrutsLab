### 画面レイアウト

7条件複合検索（発生日×設備種別×異常種別×ステータス×重大度×担当班×キーワード）。検索条件保存/呼出機能。一覧：報告番号、発生日時、設備名、異常種別、重大度（色分け）、ステータス。ページング。「一括ステータス更新」「CSV出力」「PDF出力」ボタン。

### テスト難点

- 7条件複合検索の全組み合わせ検証困難
- 検索条件保存→呼出の再現性
- PDF出力の新規ウィンドウハンドル

## 異常報告登録 (SCR-INC-CREATE-001)

| 項目 | 値 |  |  |  |  |

| 画面名 | 異常報告登録 |  |  |  |  |
| プログラムID | PG-INC-CREATE-001 |  |  |  |  |
| サブシステム | 異常報告 |  |  |  |  |

### 画面レイアウト

3ブロック構成。①発生情報：点検実施からデータ引継ぎ。②異常内容：異常種別、重大度、異常部位、異常詳細、添付ファイル（最大3枚）。③暫定処置。「類似事例検索」ボタン→同画面下部に結果表示（DOM動的追加）。「登録」「一時保存」ボタン。

### テスト難点

- 点検実施→異常報告のデータ引継ぎ（hidden/セッション）
- 類似事例検索のDOM動的追加（同画面内）
- 添付ファイル＋バリデーション同時処理

## 異常報告詳細 (SCR-INC-DETAIL-001)

| 項目 | 値 |  |  |  |  |

| プログラムID | PG-INC-DETAIL-001 |  |  |  |  |
| サブシステム | 異常報告 |  |  |  |  |

### 画面レイアウト

3ブロック表示＋経過記録タイムライン。多段階ステータス遷移：未了→調査中（調査開始）→対応中（対応開始、推定原因必須）→完了（完了、対応内容必須）→クローズ/再発防止策登録。現在ステータスにより押せるボタンが変化。

### テスト難点

- 多段階ステータス遷移（5段階）の全パス網羅
- 各遷移時の条件付き必須項目（推定原因/対応内容）
- ボタン表示/非表示の状態別切替
- タイムライン時系列ソート検証

### 画面レイアウト

4ステップWizard：①対象設備選択 → ②点検項目テンプレート選択 → ③日程・担当者設定 → ④確認・登録。ステップ間データはhiddenで持ち回り。各ステップに「戻る」「進む」。確認画面に「一時保存」「確定」。

### テスト難点

- 4ステップのデータ持ち回り（hidden→再POST）
- 「戻る」→修正→「進む」ループの全パス網羅
- ステップ3まで進んでステップ1不備発覚→戻る→修正→再確認
- 一時保存→セッション切れ→再開不可

| 画面ID | SCR-INS-EXEC-001 |  |  |  |  |

| サブシステム | 点検計画・実施 |  |  |  |  |

### 画面レイアウト

3ブロック構成。①設備基本情報（表示のみ）。②チェックリスト：大分類→中分類→項目のネストテーブル。各項目に判定radio（○/×/△）、実測値、所見。判定×→実測値+所見必須、△→所見必須。写真添付（最大5枚/項目）。③総合判定、総合所見。総合判定「異常あり」→「異常報告へ」ボタン表示。

### テスト難点

- ネストテーブル：trの親子関係がDOM上フラット
- 条件付きバリデーションの組み合わせ爆発（全項目×3状態=3^N通り）
- 写真添付＋同時バリデーション
- 「異常報告へ」ボタンの条件表示/非表示

### 画面レイアウト

実施済み点検結果の全ブロック表示（読取専用）。「修正申請」ボタン→修正理由（textarea, 必須）→承認申請。申請後は承認されるまで再修正不可。

### テスト難点

- 修正理由バリデーション（申請時のみ必須）
- 承認済み後の修正不可検証

- 操作後通知メッセージ表示
- ページ跨ぎ選択不可

## 点検実施入力 (PG-INS-EXEC-001)

| 項目 | 値 |
| --- | --- |
| プログラムID | PG-INS-EXEC-001 |

| サブシステム | 点検実施 |
| 版数 | 1.0 |
| 作成日 | 2026/05/23 |

### プログラム概要

ネストされたチェックリスト形式で点検結果を入力。判定値（○/×/△）に応じた条件付きバリデーション。写真添付（最大5枚/項目）。総合判定「異常あり」→異常報告へデータ引継ぎ。

### 入出力定義

| 区分 | 項目名 | 型 | Formプロパティ | 必須 | 備考 |
| --- | --- | --- | --- | --- | --- |
| 入力 | 判定[{i}] | String[] | execJudge | ○ | radio: ○/×/△ |
| 入力 | 実測値[{i}] | String[] | execValue | ×時必須 |  |
| 入力 | 所見[{i}] | String[] | execNote | ×/△時必須 |  |
| 入力 | 写真[{i}][{j}] | FormFile[][] | execPhoto |  | max 5枚/項目 |
| 入力 | 総合判定 | String | summaryJudge | ○ | radio |
| 入力 | 総合所見 | String | summaryNote |  |  |

### 処理詳細（Action.execute フロー）

| No. | 処理内容 | 関連DAO | 備考 |
| --- | --- | --- | --- |
| 1 | 点検項目マスタから3階層チェックリスト構築 | ChkItemDao.findByTmpl() |  |
| 2 | 各項目の判定値に応じたバリデーション |  | Validator + Action内追加検証 |
| 3 | ×→実測値+所見必須、△→所見必須、○→任意 |  |  |
| 4 | 写真保存: 項目{i}/写真{j}→filesystem |  | /attachments/ins/{planId}/{itemIdx}/ |
| 5 | 総合判定「異常あり」→異常報告登録用データをセッションへ |  | Forward: inc.create |

### 入力検証ルール

| No. | 対象項目 | 検証ルール | エラーメッセージ（ja） |
| --- | --- | --- | --- |
| 1 | execJudge[{i}] | 空不可 | 判定を選択してください |  |

| 3 | execNote[{i}] | ×/△時: 空不可 | ×/△判定の場合は所見を入力してください |  |

| 区分 | 項目名 | 型 | Formプロパティ | 必須 | 備考 |
| --- | --- | --- | --- | --- | --- |
| 入力 | 修正理由 | String | modifyReason | 申請時○ |  |
| 出力 | 実施結果全データ | ExecResultDto | execResult |  |  |

### 処理詳細（Action.execute フロー）

| No. | 処理内容 | 関連DAO | 備考 |
| --- | --- | --- | --- |
| 1 | 指定点検IDの全データ取得 | ExecDao.findById() |  |
| 2 | 修正申請: 理由必須→approval_status=申請中に更新 | ExecDao.requestModify() |  |
| 3 | 承認済みデータは修正不可（申請ボタン非表示） |  |  |

### 入力検証ルール

| No. | 対象項目 | 検証ルール | エラーメッセージ（ja） |
| --- | --- | --- | --- |

| 遷移元 | 遷移先 | トリガー |
